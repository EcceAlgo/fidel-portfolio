## 1. Glossaire

| Terme | Définition |
|--------|------------|
| ADR (Architecture Decision Record) | Document permettant de consigner les principales décisions d'architecture prises au cours du projet. |
| API | Interface permettant à deux applications de communiquer entre elles. |
| CI/CD | Ensemble des pratiques d'intégration continue et de déploiement continu visant à automatiser les phases de validation et de mise en production. |
| DevOps | Approche visant à rapprocher les équipes de développement et d'exploitation afin d'améliorer la qualité et la rapidité des livraisons. |
| DevSecOps | Extension du DevOps intégrant la sécurité tout au long du cycle de développement logiciel. |
| Git | Système de gestion de versions distribué permettant de suivre l'évolution du code source. |
| GitHub | Plateforme d'hébergement de dépôts Git facilitant la collaboration et l'automatisation des projets. |
| MVP (Minimum Viable Product) | Première version du produit intégrant uniquement les fonctionnalités essentielles. |
| Portfolio | Application web destinée à présenter le profil, les compétences et les réalisations d'un développeur. |
| Pull Request (PR) | Demande de fusion de modifications d'une branche vers une autre après relecture et validation. |
| Responsive Design | Technique permettant à une interface de s'adapter aux différentes tailles d'écran. |
